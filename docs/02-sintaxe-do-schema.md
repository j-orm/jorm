# Step 2: Schema Syntax ✍️

Jorm uses its own declarative language (`.jorm` files) that serves as the **single source of truth** for both the database and the generated Java classes. In this guide, you will learn all the concepts needed to write comprehensive schemas.

Open the `.jorm/schema.jorm` file and let's explore.

---

## 1. The Configuration Block

The `config` block defines the global options for your project: which database to use, where to generate the code, and what framework to use.

```jorm
config {
    database  = "postgresql"  // Or: mysql
    output    = "src/main/java/generated"
    package   = "com.myproject.db"
    framework = "spring"      // Optional: adds @Repository to the generated client
}
```

| Key | Required | Description |
| --- | --- | --- |
| `database` | Yes | SQL Dialect. Values: `postgresql` or `mysql` |
| `output` | Yes | Path to the folder where the code will be generated |
| `package` | Yes | Java package of the generated files |
| `framework` | No | If `spring`, adds `@Repository` to the client |

---

## 2. Defining Enums

Enums are mapped to the `ENUM` type in PostgreSQL (or `VARCHAR` in MySQL) and become a native `enum` in Java.

```jorm
enum Role {
    USER
    ADMIN
    MODERATOR
}
```

The generated Java enum will be:

```java
public enum Role {
    USER, ADMIN, MODERATOR
}
```

---

## 3. Defining Models (Tables)

The `model` keyword represents a database table and an immutable `Record` in Java.

```jorm
model User {
    id        Int      @id @autoincrement
    email     String   @unique
    name      String
    role      Role
    isActive  Boolean
    createdAt DateTime
    posts     [Post]   @relation(fields: [id], references: [authorId])
}

model Post {
    id        Int      @id @autoincrement
    title     String
    content   String
    authorId  Int
    author    User     @relation(fields: [authorId], references: [id])
}
```

---

## 4. Supported Data Types

| Jorm Type | Java Type | SQL Type (PostgreSQL) | SQL Type (MySQL) |
| --- | --- | --- | --- |
| `Int` | `Integer` | `INTEGER` / `SERIAL` | `INT` / `AUTO_INCREMENT` |
| `String` | `String` | `VARCHAR(255)` | `VARCHAR(255)` |
| `Float` | `Float` | `DOUBLE PRECISION` | `DOUBLE` |
| `Boolean` | `Boolean` | `BOOLEAN` | `TINYINT(1)` |
| `DateTime` | `LocalDateTime` | `TIMESTAMP` | `DATETIME` |

---

## 5. Available Annotations

| Annotation | Applies to | Description |
| --- | --- | --- |
| `@id` | Field | Defines the primary key of the table |
| `@autoincrement` | `Int` Field | The column is automatically generated by the DB |
| `@unique` | Field | Adds the `UNIQUE` constraint to the column |
| `@relation(...)` | Model Field | Defines a foreign key between models |

---

## 6. Relations between Models

Relations are defined using `@relation`. The field that stores the foreign key uses `fields` and `references` to indicate which local column points to which remote column.

```jorm
model Post {
    id       Int    @id @autoincrement
    authorId Int
    author   User   @relation(fields: [authorId], references: [id])
}
```

In this example, `authorId` is the foreign key column and `id` is the referenced column in the `User` table.

> **Note:** Full support for complex and bidirectional relationships is part of Jorm's roadmap.

## 7. Complete Schema Example

A real-world schema with config, enum, and two related models:

```jorm
config {
    database  = "postgresql"
    output    = "src/main/java/generated"
    package   = "com.myproject.db"
    framework = "spring"
}

enum Role {
    USER
    ADMIN
}

model User {
    id        Int      @id @autoincrement
    email     String   @unique
    name      String
    role      Role
    isActive  Boolean
    createdAt DateTime
    posts     [Post]   @relation(fields: [id], references: [authorId])
}

model Post {
    id        Int    @id @autoincrement
    title     String
    content   String
    authorId  Int
    author    User   @relation(fields: [authorId], references: [id])
}
```
